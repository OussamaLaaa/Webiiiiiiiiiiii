# دليل النشر اليدوي على Vercel

## 📋 نظرة عامة

بما أن هناك مشاكل في الصلاحيات مع GitHub، سنقوم بنشر الموقع يدوياً من خلال واجهة Vercel. هذا الدليل يشرح الخطوات بالتفصيل.

## 🚀 خطوات النشر

### الخطوة 1: تسجيل الدخول إلى Vercel

1. اذهب إلى [vercel.com](https://vercel.com)
2. سجل الدخول باستخدام حسابك
3. إذا لم يكن لديك حساب، قم بإنشاء حساب جديد

### الخطوة 2: إنشاء مشروع جديد

1. بعد تسجيل الدخول، اضغط على زر **"Add New..."** في الزاوية اليمنى العليا
2. اختر **"Project"**

### الخطوة 3: استيراد المشروع من GitHub

1. في قسم **"Import Git Repository"**، ابحث عن مستودعك:
   - `OL-guuuu/Webiiiiiiiiiiii.git`
   
2. إذا لم يظهر المستودع:
   - اضغط على **"Adjust GitHub App Permissions"**
   - تأكد من أن Vercel لديه صلاحيات للوصول إلى المستودع
   - أو استخدم خيار **"Import from Git"** وأدخل رابط المستودع يدوياً

### الخطوة 4: إعدادات المشروع

Vercel سيكتشف تلقائياً إعدادات المشروع:

```
Framework Preset: Vite
Build Command: npm run build
Output Directory: dist
Install Command: npm install
```

تأكد من أن هذه الإعدادات صحيحة، ثم اضغط **"Deploy"**

### الخطوة 5: إعداد المتغيرات البيئية

بعد اكتمال النشر الأول:

1. اذهب إلى **Settings** → **Environment Variables**
2. أضف المتغير التالي:
   - **Name**: `DASHBOARD_PASSWORD`
   - **Value**: `00000008` (أو كلمة مرور قوية أخرى)
3. اضغط **"Save"**

### الخطوة 6: إعادة النشر

بعد إضافة المتغيرات البيئية:

1. اذهب إلى **Deployments**
2. اضغط على زر **"..."** بجانب آخر نشر
3. اختر **"Redeploy"**
4. تأكد من أن **"Build Environment Variables"** محددة
5. اضغط **"Redeploy"**

## 🔗 روابط المشروع

بعد اكتمال النشر، ستحصل على:

- **رابط الموقع**: `https://webiiiiiiiiiiiiiii.vercel.app`
- **رابط الـDashboard**: `https://webiiiiiiiiiiiiiii.vercel.app/dashboard`
- **رابط API**: `https://webiiiiiiiiiiiiiii.vercel.app/api/config`

## ✅ التحقق من النشر

### 1. التحقق من الموقع العام

افتح رابط الموقع في المتصفح وتأكد من:
- تحميل الصفحة بنجاح
- عرض المحتوى بشكل صحيح
- عمل جميع الأنيميشن

### 2. التحقق من الـDashboard

1. افتح رابط الـDashboard
2. أدخل كلمة المرور: `00000008`
3. تأكد من:
   - تحميل الـDashboard بنجاح
   - إمكانية تعديل الإعدادات
   - حفظ التغييرات

### 3. التحقق من API

يمكنك التحقق من API باستخدام curl:

```bash
# جلب الإعدادات
curl https://webiiiiiiiiiiiiiii.vercel.app/api/config

# تحديث الإعدادات (مع المصادقة)
curl -X PUT https://webiiiiiiiiiiiiiii.vercel.app/api/config \
  -H "Authorization: Bearer 00000008" \
  -H "Content-Type: application/json" \
  -d '{"siteName":"Test Site"}'
```

## 🔄 النشر التلقائي

بعد إعداد المشروع لأول مرة، سيتم نشره تلقائياً عند كل دفع جديد إلى GitHub:

1. قم بتعديل الملفات محلياً
2. ادفع التغييرات إلى GitHub:
   ```bash
   git add .
   git commit -m "Your commit message"
   git push origin main
   ```
3. Vercel سيكتشف التغييرات وينشرها تلقائياً

## 🛠️ استكشاف الأخطاء

### المشكلة: فشل البناء

**الحل:**
1. تحقق من سجلات البناء في Vercel
2. تأكد من أن جميع التبعيات مثبتة
3. تحقق من أن `package.json` يحتوي على جميع السكريبتات المطلوبة

### المشكلة: API لا يعمل

**الحل:**
1. تأكد من أن `api/config.js` موجود في المجلد الصحيح
2. تحقق من أن المتغيرات البيئية مضبوطة
3. راجع سجلات الدوال في Vercel

### المشكلة: الـDashboard لا يحفظ التغييرات

**الحل:**
1. تأكد من أن `DASHBOARD_PASSWORD` مضبوط في Environment Variables
2. تحقق من أن API يعمل بشكل صحيح
3. راجع سجلات المتصفح (Console) للأخطاء

### المشكلة: الموقع لا يحدث تلقائياً

**الحل:**
1. تأكد من أن `SiteConfigContext` يجلب الإعدادات من API
2. تحقق من أن `useEffect` يعمل بشكل صحيح
3. راجع سجلات الشبكة (Network) في المتصفح

## 📊 مراقبة الأداء

يمكنك مراقبة أداء الموقع من خلال:

1. **Analytics**: في Vercel Dashboard
2. **Logs**: سجلات البناء والوظائف
3. **Speed Insights**: تحليل سرعة الموقع

## 🔒 الأمان

### نصائح مهمة:

1. **غير كلمة المرور**: استخدم كلمة مرور قوية في الإنتاج
2. **استخدم HTTPS**: Vercel يوفر HTTPS تلقائياً
3. **حدد الوصول**: يمكنك إضافة قواعد وصول إضافية في `vercel.json`
4. **راقب السجلات**: راقب سجلات API للأنشطة المشبوهة

## 🎯 الخطوات التالية

بعد النشر الناجح:

1. **اختبار الموقع**: تأكد من أن كل شيء يعمل بشكل صحيح
2. **إعداد النطاق المخصص**: يمكنك ربط نطاق خاص بك
3. **تحسين الأداء**: استخدم Vercel Analytics لتحسين الأداء
4. **إضافة قاعدة بيانات**: للإنتاج، استخدم قاعدة بيانات حقيقية بدلاً من in-memory storage
